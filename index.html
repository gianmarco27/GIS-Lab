Data Collection

We have collected the data for each segment of the road leaving out those attributes that were almost absent in Niguarda (Buffer, Bike lane) and those that were constant throughout the duration of the survey (number of lanes, parking slots).

Problems Encountered

  • Multipath

    Many collected points had too poor accuracy and needed to be relocated, due to both the multipath effect and the fact that all the surveyling had to be performed with mobile phone sensors.

  • Discontinuous Fragments

    Often small buffers and some data were discontinuous, in the same road id we could have discrepancies and therefore decide which data to keep for PLOS elaboration.

Data Processing

To process all the data, different steps were taken both via QGIS operations and Python scripts.

Data Elaboration and Styling

Since it was our first time working on this kind of data, some sorts of adjustments to the data was required to perform the computation and rapresentation, the roadlinks provided for Niguarda were divided in different segments with sometimes different 'source_id's near intersections and crossings, this was unknown at the time of surveyling and therefore some points data had to be taken from the road these fragments belonged to.
In addiction to this, since Qgis was unstable while performing some operations of data matching, we used some python scripts to generate tables for PLOS computation. During the creation of the WEB-GIS to perform the desired styling and customization some trial and error had to be performed to correctly make use of OpenLayers provided tools.

Final Result

All the data are stored in the repository Geoserver folder
This allows us to execute the WEB-GIS from any PC the repository gets cloned to.
